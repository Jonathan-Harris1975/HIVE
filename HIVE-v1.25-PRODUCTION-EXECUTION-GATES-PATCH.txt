HIVE v1.25 production execution gates patch

Purpose
- Remove the production blocker where approved review decisions still reported can_execute_now=false.
- Remove the blocker where controlled execution previews always reported adapter execution as disabled.
- Replace the v1.23 API-contract build marker with v1.25-production-execution-gates.

Key backend changes
- Added backend/app/services/execution_adapters.py as the shared production adapter policy surface.
- Added EXECUTION_ADAPTERS_ENABLED and EXECUTION_ADAPTERS_REQUIRE_APPROVAL settings, defaulting to true.
- /health now exposes execution_adapters_enabled and execution_adapter_policy for HIVE-UI/Ops.
- Approved execution-review decisions now record:
  - can_execute_now: true
  - adapter_execution_enabled: true
  - execution_state: ready_for_execution
  - execution_mode: approved_controlled_execution
- Controlled execution previews now show approved adapter handoff as ready_for_execution instead of blocked.
- Workflow simulations now remove execution_adapter_layer_not_enabled from approved/default production paths.
- Shared skill execution plans now describe review-gated production handoff rather than future-disabled adapters.

Guardrail retained
- Approval unlocks the operator-triggered allow-listed adapter handoff.
- The approval decision endpoint does not auto-run repo pushes, package installs or background jobs.

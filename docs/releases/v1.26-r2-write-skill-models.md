# HIVE v1.26 R2 write, skill and model selection update

This release moves HIVE from read-mostly R2 browsing to production-controlled read/write storage operations across configured ecosystem lanes.

## Storage

- Upload object keys are now human-readable and date-based, keeping the original filename visible.
- Multipart, text and base64 upload endpoints accept a target `lane`.
- Configured lanes are marked read/write when shared server-side R2 write credentials allow it.
- `GET /v1/files/r2/{lane}/view?key=...` streams objects with inline content disposition for browser viewing.

## Skills

- `POST /v1/skills/from-file` registers an uploaded file as a searchable skill descriptor in D1.
- The endpoint records source lane, object key, public URL, tags, repo, risk and priority metadata.

## Models

- Image, video, audio and infrastructure model categories are selectable rather than discovery-only.
- Non-text model selections carry an advisory instead of being blocked by the model router.

## Production readiness

- Shared R2 write credentials satisfy multi-bucket read readiness.
- Required R2 lanes remain checked through the configured lane registry.

# v1.27 - Repository Manager (Phase 1: Repository Intelligence)

## Summary

Introduces the Repository Manager: the first phase of the HIVE production
upgrade programme. Uploaded repository ZIPs are safely extracted into a
per-process temporary working directory, fingerprinted, and reduced to a
manifest (file count, size, detected languages, dependency manifests found).
Repositories are never persisted permanently on local disk; automatic
cleanup removes idle working copies after `REPOSITORY_TTL_SECONDS`.

## New capabilities

- Safe ZIP extraction reusing the existing `app.ingestion.zip_ingestion`
  path-traversal guards (no duplicate extraction logic).
- Repository fingerprinting: SHA-256 over the sorted (path, content-hash)
  file index.
- Manifest generation: file/byte counts, per-language file counts, and
  best-effort dependency manifest detection (`requirements.txt`,
  `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `composer.json`,
  `Gemfile`, `Pipfile`).
- Incremental re-indexing: `reindex_repository` and `repository_diff`
  report added/changed/removed files against the last indexed snapshot
  without re-extracting the archive.
- In-process repository registry (`list_repositories`, `get_repository`)
  with explicit (`cleanup_repository`) and TTL-based
  (`cleanup_expired_repositories`) cleanup.

## New endpoints (Bearer-authenticated)

| Endpoint | Purpose |
|---|---|
| `POST /v1/repositories` | Upload a ZIP and register a repository |
| `GET /v1/repositories` | List registered repositories |
| `GET /v1/repositories/{id}` | Current manifest for a repository |
| `GET /v1/repositories/{id}/diff` | Preview added/changed/removed files |
| `POST /v1/repositories/{id}/reindex` | Recompute the manifest incrementally |
| `DELETE /v1/repositories/{id}` | Remove a repository and its temp files |
| `POST /v1/repositories/cleanup` | Sweep idle repositories past the TTL |

## Validation performed in this sandbox

- `python -m py_compile` on all new/modified files (clean).
- Manual trace of every code path in `repository_manager.py` against the
  new test suite (extraction, unsafe-ZIP rejection, fingerprinting,
  language detection, dependency parsing, incremental diff/reindex,
  registry listing, explicit cleanup, TTL-based cleanup).
- Confirmed no test in `backend/tests` hard-codes the R2 ecosystem lane
  count or lane set, so the new `repositories` lane is additive-safe.

**Not run here:** `pytest`, `ruff`, and a live app import/build. This
sandbox has no network access, so `pip install -r requirements.txt` cannot
fetch `fastapi`/`pydantic-settings`/`boto3`/etc., and there is no way to
exercise the real test/lint toolchain. Before merging, run the standard
sequence locally or in CI:

```bash
python -m pip install -r requirements.txt -r requirements-dev.txt
PYTHONPATH=backend python -m pytest backend/tests -q
python -m ruff check backend/app backend/tests scripts
```

## Deferred to later phases

- Persisting manifests/fingerprints in D1/SQL (Phase 2, Repository Memory).
- Uploading extracted archives to the `hive-repositories` R2 bucket rather
  than relying purely on local temp storage (Phase 9, Bucket Manager).
- A background scheduler calling `cleanup_expired_repositories` on a timer
  (currently exposed only as `POST /v1/repositories/cleanup`, callable from
  MAST or a cron-style ops job).

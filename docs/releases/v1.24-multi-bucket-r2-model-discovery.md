# HIVE v1.24: multi-bucket R2 access and model discovery

This release extends the existing v1.23 production backend without changing the owner-frozen chat persistence, database, version or historical test files.

## R2 access boundary

The primary `uploads` lane continues to use the existing read/write credential pair:

- `R2_ACCESS_KEY_ID`
- `R2_SECRET_ACCESS_KEY`

All other configured R2 lanes use the separate read-only credential pair:

- `R2_MULTI_BUCKET_READ_ENABLED=true`
- `R2_READ_ACCESS_KEY_ID`
- `R2_READ_SECRET_ACCESS_KEY`

The backend never exposes either credential to the browser. Non-upload lanes have no upload, overwrite or delete route.

## New authenticated endpoints

```text
GET /v1/files/r2/{lane}/objects
GET /v1/files/r2/{lane}/metadata
GET /v1/files/r2/{lane}/read
GET /v1/files/r2/{lane}/download
```

The object-list endpoint supports prefix browsing, delimiter-based folder discovery, bounded filename search, pagination cursors and an operator-configurable scan ceiling.

`POST /v1/chat/with-file` now accepts a `lane` field. The uploads lane may use persisted SQL/Vectorize chunks. Other lanes use a bounded direct read and remain read-only.

## Model discovery

`GET /v1/models` now returns safe picker metadata:

- configured HIVE roles;
- free-model detection;
- input and output modalities;
- grouped categories;
- standard-chat eligibility;
- a human-readable disabled reason.

Image-generation and video-generation models are discovery-only in this release. They are visible to HIVE-UI but cannot be selected for ordinary chat until a dedicated creation workspace is added.

## Production controls

Additional environment variables:

```text
R2_MULTI_BUCKET_READ_ENABLED=false
R2_READ_ACCESS_KEY_ID=
R2_READ_SECRET_ACCESS_KEY=
R2_MULTI_BUCKET_MAX_SCAN_KEYS=5000
R2_DOWNLOAD_MAX_BYTES=536870912
```

When multi-bucket reading is enabled in production, startup readiness fails if either read-only R2 credential is missing.

## Verification

The release adds isolated v1.24 tests for:

- read/write versus read-only lane policy;
- scoped credential selection;
- prefix listing, metadata, preview and streaming download;
- non-upload file chat;
- production readiness validation;
- model grouping and discovery-only media generation.

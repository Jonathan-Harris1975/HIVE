# HIVE v1.17 Registry Integrity

Build marker: `v1.22-workflow-simulation-persistence`

v1.17 is a consolidation and trust release. It keeps the v1.16 skill-search, recommendation, routing, review-queue and evidence-pack features, then adds read-only registry integrity checks so HIVE can validate the imported shared skills catalogue before later recommendation/routing work becomes more active.

## Added endpoints

```text
GET  /v1/skills/integrity
GET  /v1/skills/duplicates
GET  /v1/skills/missing
GET  /v1/skills/orphans
POST /v1/skills/rebuild-index
```

## What the integrity report checks

- Duplicate skill IDs.
- Duplicate slugs.
- Duplicate object keys.
- Duplicate search-document IDs.
- Missing required metadata fields.
- Invalid priority tiers.
- Invalid risk levels.
- Invalid repo mappings.
- Descriptor URL/object-key mismatches.
- Wrong D1 lane/source-type records.

## Safety model

All new checks are read-only except `/v1/skills/rebuild-index`, which is a guarded wrapper around the existing R2 search-document importer. Its default is `dry_run:true` and live mode only upserts D1 metadata. It does not mutate R2 descriptors, install skills, execute tools, or alter repos.

## Smoke test

```bash
ADMIN_BEARER_TOKEN=... python scripts/v117_registry_integrity_smoke.py
```

## Expected healthy result

A clean registry should return something close to:

```json
{
  "ok": true,
  "checked_count": 201,
  "issue_count": 0,
  "registry_health": 100
}
```

If issues are found, treat the output as a review list. v1.17 intentionally reports drift rather than repairing or deleting anything automatically.
